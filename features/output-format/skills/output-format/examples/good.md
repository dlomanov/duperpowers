# Good Examples - Topdown Chat Responses

Each example is a realistic chat response. Notice throughout: full sentences,
islands separated by blank lines, lowercase children, no trailing periods,
`=>` marks causality and conclusions.

## case: explaining a technical concept, with diagram

Лидер в Raft не может закоммитить чужую запись простым подсчетом реплик
  запись из прошлого терма могла разъехаться по узлам до смены лидера
  новый лидер видит ее на большинстве – но это не доказывает финальность
    узел с более свежим логом мог вообще не получить эту запись
  поэтому лидер коммитит подсчетом реплик только записи своего терма
    чужие записи доезжают следом – они лежат в логе ниже новой точки коммита
      term 2: [x=1]  реплицирована на 2 из 3, не закоммичена
      term 3: лидер сменился, пишет [y=2]
              коммит [y=2] => [x=1] закоммичена автоматически
=> правило: считай реплики только для своего терма, остальное приедет само

  тесты пакета зеленые – кроме TestRetryBudget
    он парсит строки старого формата лога и падает на новом

Два варианта закрыть

## case: debugging story - hypothesis, evidence, conclusion

Нашел причину дабл-чарджа – это не ретрай, это два консумера
  гипотеза про ретрай отпала: idempotency-ключ в обоих логах одинаковый и сработал

## case: digest of someone's notes - synthesis over echo

Твои заметки про хайлайт противоречат твоей же ALARM-системе
  хайлайт работает через тоннель – hyperfocus и не выдергивать себя
  но: ALARM каждые 90 минут – это анти-тоннель, он выдергивает принудительно
  оба написаны в один период => ты еще не решил, тоннель друг или враг

Моя ставка – тоннель в важную задачу друг, тоннель в игры враг
  значит будильник нужен не по расписанию, а по типу активности

## case: decision with trade-offs

Для локов поверх Postgres есть два честных пути – и один выбор
  advisory locks: дешево, нативно, но лок живет внутри сессии
    упала сессия => лок исчез, для долгих задач это фича и ловушка сразу
  таблица локов с lease: переживает реконнект, видна в SQL, но нужен фоновый продлевальщик
    и придется самому чистить протухшие записи
  у тебя воркеры держат задачи по 10+ минут через пулер
    пулер рвет сессии => advisory locks отпадают по построению
=> таблица с lease – дороже в коде, но единственная переживает твой пулер

## case: socratic question that opens reasoning

Почему кэш на чтение не спасет этот эндпоинт?
  профиль показывает: 80% времени – сериализация ответа, а не запрос в базу
  кэшировать готовый json? тогда встает инвалидация
    цены меняются каждые ~30 секунд => TTL короче окна попадания
  => кэш будет попадать редко и съедать память – лечить надо сериализацию

## case: multi-topic response with referenceable ids

When several islands answer at once, each root gets a T-id so the user can reply
"T1 согласен, T3 переделай" without quoting.

T1 Миграция логгера готова, держит только TestRetryBudget
  тест парсит старый формат лога и падает на slog
  правлю парсер на буфер-хендлер => мердж сегодня

T2 Кэш в checkout не взлетит – проблема не в базе
  80% времени уходит на сериализацию, а не на запрос
  кэшировать json упрется в инвалидацию каждые 30 секунд

T3 Дейли предлагаю двигать на 12
  это открывает 3 часа утреннего глубокого блока до инбаунда
  но: нужно согласовать с Олегом, у него стендап в 11

T4 Что от тебя нужно
  T1 закрываю сам, без тебя
  T2 и T3 жду твое слово – двигать или нет
=> ответь по айди, дальше разрулю

(a conclusion across all topics is its OWN island T4, not a => hung off T3;
the => inside T4 concludes only T4)

## case: a plan with numbered blocks

Numbered steps are the one place markers are allowed - a plan is a sequence,
and the island sits as elaboration under each step.

План на завтра собран из трех блоков
  1. B1 50/10 – миграция логгера, тест TestRetryBudget переписать на slog
       это держит мердж со вчера => первым делом
  2. B2 50/10 – ресерч text2speech-тулов для будильников
       только сбор вариантов, без выбора – выбор отдельным блоком
  3. побегать в парке – без таймера, это не рабочий блок
=> старт в 10:00 с B1, пока голова свежая и Slack молчит
